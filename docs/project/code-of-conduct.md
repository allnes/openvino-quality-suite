# Code of conduct

OVIQS follows the repository-level `CODE_OF_CONDUCT.md`.

Contributors should keep technical discussion respectful, specific and focused on
the work.

## Review behavior

Good technical disagreement is expected. Keep comments tied to concrete behavior,
tests, docs, contracts or maintenance risk.

Avoid personal remarks, vague criticism and pressure to bypass the quality gate.
If a discussion stalls, summarize the disputed technical point and propose the
smallest reproducible check that would settle it.

## Scope

This applies to repository discussions, issues, pull requests, documentation
reviews and project communication related to OVIQS.
